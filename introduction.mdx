VainEngine is a lightweight 3D game engine written in C#, built on top of OpenTK and OpenGL. It provides a familiar **GameObject-Component architecture**, physics via the Jitter engine, audio playback, UI rendering, and scene management — everything you need to build small to medium-sized 3D games.

## Key features

<CardGroup cols={2}>
  <Card title="GameObject-Component system" icon="puzzle-piece">
    Attach reusable components to any game object, inspired by Unity's familiar pattern.
  </Card>
  <Card title="3D rendering with OpenGL" icon="display">
    Perspective and orthographic cameras, OBJ mesh loading, textures, normal maps, and emission maps.
  </Card>
  <Card title="Jitter physics" icon="bolt">
    Integrated rigid body physics with box and capsule colliders, gravity, and collision events.
  </Card>
  <Card title="Built-in FPS controller" icon="person-walking">
    Ready-to-use first-person controller with wall-running, crouching, jumping, and debug mode.
  </Card>
  <Card title="Audio playback" icon="music">
    NAudio-based audio engine supporting file playback and cached sound clips.
  </Card>
  <Card title="Scene management" icon="film">
    JSON-based scene files with runtime loading and hot-reload support.
  </Card>
</CardGroup>

## Getting started

<Steps>
  <Step title="Add VainEngine to your project">
    Reference the `VainEngine.csproj` in your solution or add a project reference from Visual Studio.
  </Step>
  <Step title="Create your game class">
    Inherit from `GameLogic` and override `Start()` and `Update()` to write game code.
  </Step>
  <Step title="Create and run the window">
    Use `Window.Create()` to open a game window, call `Setup()` with your game class, then `Run()`.
  </Step>
  <Step title="Build your scene">
    Spawn `GameObject` instances, assign meshes, add components, and configure the camera.
  </Step>
</Steps>

<Note>
VainEngine uses a **vertically flipped Y axis** compared to engines like Unity. A position of `(1, 2, 3)` in Unity is `(1, -2, 3)` in VainEngine. See the [coordinate system reference](/reference/coordinate-system) for details.
</Note>
